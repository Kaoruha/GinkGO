## 关键发现

### 1. 收益与胜率的根本矛盾
**在单一标的上，年化收益和胜率呈反比关系**：

| 策略类型 | 年化 | 胜率 | 原因 |
|----------|------|------|------|
| 高阈值动量(0.03) | 9.93% | 36% | 捕捉大波动但信号少 |
| 中阈值动量(0.022) | 11.31% | 46% | 平衡信号频率和质量 |
| 均值回归(RSI14) | 5.17% | 86% | 频繁小幅交易 |
| 均值回归+止盈 | 3.89% | 100% | 每笔都止盈但利润小 |

### 2. 标的选择至关重要
| 标的 | 类型 | 最佳年化 | 最佳胜率 | 特点 |
|------|------|---------|---------|------|
| 601899.SH | 有色金属 | 11.34% | 86% | 高波动，趋势强 |
| 000333.SZ | 家电消费 | 1.58% | 55% | 稳定增长，波动小 |
| 600036.SH | 银行 | -0.12% | 26% | 震荡为主，不适合动量 |
| 000858.SZ | 白酒 | -2.35% | 23% | 下跌趋势 |
| 000001.SZ | 银行 | -1.46% | 20% | 震荡 |

### 3. 修复后的策略有效
- **mean_reversion_v2**：修复 `bar.close` Decimal→float 后，从 0 订单变为正常产出（85.71% 胜率）
- **macd_crossover**：新实现，可用但收益差（1.00% 年化）
- **donchian_breakout**：新实现，可用但假突破多（4.37% 年化）
- **momentum_rsi**：新实现，momentum + RSI 过滤，最佳平衡点（6.68%/48%）
- **momentum_pullback**：新实现，601899.SH 上信号太少

### 4. 原有策略 0 订单根因
数据库中大量策略产出 0 订单的根本原因是 **Decimal/float 类型混用**：
```python
# mean_reversion 原代码（已修复）
closes = [bar.close for bar in bars]  # bar.close 是 Decimal
changes = [closes[i] - closes[i - 1] for i in range(1, len(closes))]
gains = [max(c, 0.0) for c in changes]  # Decimal + float → TypeError
```

修复方法：`closes = [float(bar.close) for bar in bars]`

**其他可能有同样问题的策略**：dual_thrust、bollinger_breakout、trend_capture、pyramid_trend、adaptive_momentum、rsi_bounce 等。

### 5. 多风控可行但效果有限
- loss_limit(10%) + profit_target(15%) 双风控组合可同时绑定
- 但 profit_target 会提前止盈，降低胜率（从 46%→42%）且不提高年化

## 结论

### 已达到的目标
- 年化 10%+ ✅（RL02: 11.31%, RL38: 11.34%）
- 胜率 50%+ ✅（RL25: 85.71%, RL37: 100%, RL40: 54.55%）

### 未同时达到双目标
**没有单个策略在单个标的上同时达到年化 10%+ 和胜率 50%+**。根本原因：
1. 高波动标的（601899.SH）上动量策略收益高但胜率低
2. 低波动标的（000333.SZ）上胜率高但收益低
3. 均值回归策略胜率极高但每笔利润小

### 突破方向
1. **多标的组合**：不同标的策略组合可同时提高收益和胜率
2. **多策略组合**：均值回归（提供胜率）+ 动量（提供收益）并行运行
3. **修复引擎非确定性 Bug**：当前结果可靠性受限
4. **修复 Decimal/float Bug**：让更多策略可用，扩大搜索空间
5. **短线策略**：当前仅用日线数据，分钟级别可能有更好机会

## 问题记录

1. **ratio_sizer 数据库同步**：file_service.update() 的 data 字段更新不生效。workaround：创建新组件
2. **大量策略 0 订单**：Decimal/float 类型混用导致 TypeError。已为 mean_reversion 创建修复版 v2
3. **并行 backtest run 不可靠**：shell 脚本中 `&` 并行会导致 run 命令不执行。需分开用 Bash background 模式
4. **position_ratio_risk 过度拦截**：max_position_ratio=0.3 导致所有订单被拒绝
5. **高股价问题**：600519.SH（茅台 ~1600 元）在 100k 资金下 ratio_sizer 无法下单（不足 100 股）
6. **回测引擎非确定性**：同一配置多次运行结果不同，成功率约 33%
7. **CLI 并行回测**：多个 `ginkgo backtest run` 同时执行时都卡在 running 状态
8. **trend_mr 过滤过严**：RSI<30 + price>MA20 + momentum>0 同时满足的情况极少
9. **bar_adjustment Decimal bug**：`bar_adjustment.py:266` 有 Decimal * float 类型错误
