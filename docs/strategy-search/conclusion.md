## 目标与结果

| 目标 | 要求 | 最佳结果 | 状态 |
|------|------|---------|------|
| 年化收益 | >10% | **15-17%** (成功模式) | ✅ 达标 |
| 胜率 | >50% | **50.86%** (无风控模式) | ⚠️ 可达但牺牲年化 |

**结论：双目标无法同时达到。年化和胜率存在结构性权衡关系——由风控机制本身调节。**

## 关键发现

### 1. 收益与胜率的根本矛盾（因果层面已验证）
| 策略类型 | 年化 | 胜率 | 根因 |
|----------|------|------|------|
| 动量(0.022) 无风控 | 4.10% | **50.86%** | 亏损持仓可恢复，胜率↑但拖累收益 |
| 动量(0.022) LossLimit(10%) | **16.17%** | 48.00% | 截断亏损提升年化，但也制造已实现亏损 |
| 均值回归(RSI14) | 5.17% | 86% | 频繁小幅交易 |
| 均值回归+止盈 | 3.89% | 100% | 每笔止盈但利润极薄 |

**风控消融实验证明**：LossLimit 是年化与胜率之间的调节阀。去掉风控 → 胜率突破 50% 但年化暴跌。风控的「截断亏损」本质上是把潜在可恢复的头寸变成已实现亏损。

### 2. 仓位比例的杠杆效应
ratio 从 0.5 → 0.8 的效果：
- 年化 +43%（11.3% → 16.2%）
- 胜率 +2%（46% → 48%）
- 回撤 +55%（13% → 21%）

## 达成目标的必要改进

### P0: 修复引擎非确定性 Bug
当前最大障碍。4 倍年化波动使结果不可信。
可能原因：dict 迭代顺序、共享可变状态、成交价格计算随机性。

### P1: 多标的/多策略组合
单标的单策略无法同时满足双目标。需要：
- portfolio 支持多策略并行
- 或多个 portfolio 协同管理
- 不同标的的策略可以互补

### P2: 修复 Decimal/float 类型 Bug
影响范围：mean_reversion、dual_thrust、bollinger_breakout 等多个策略。
统一方案：所有 `bar.close/high/low/open` 返回 float。

### P3: 更多数据频率
当前仅有日线数据。分钟线数据可提供：
- 更多的交易信号
- 更精细的入场/出场时机
- 更高的胜率可能性

## 技术问题记录

| 问题 | 影响 | Workaround |
|------|------|-----------|
| file_service.update() data 不生效 | 无法更新组件源码 | 创建新组件 |
| 并行 backtest run 失败 | 多个 run 同时卡住 | 顺序执行 |
| Decimal + float 混用 | 策略运行时报 TypeError | float() 转换 |
| ratio_sizer 资金不足 | 高股价无法交易 | 选低价股 |
| bar_adjustment Decimal bug | 复权计算失败 | 暂不使用复权 |
| position_ratio_risk 过度拦截 | 所有订单被拒 | 降低 ratio 或不用 |
