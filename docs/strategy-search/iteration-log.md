## 搜索过程总结

### 最佳标的：紫金矿业 (601899.SH)
- 价格：~34元/股，适合大仓位操作
- 波动性：强趋势+适当震荡，适合动量策略
- 其他测试标的：茅台(太贵)、招行(银行震荡)、平安(波动小)、美的(中等)、比亚迪(不适合趋势策略)、赛力斯(亏损)

### 最佳策略类型：短期动量 (Momentum)
- 回溯期 8-10天最优
- 动量阈值 2%最优
- 较短的回溯期捕捉更强的短期趋势信号
- 年化13.3%是最高记录

### 胜率分析
- 动量策略胜率普遍在 32-46% 区间
- 高胜率组合（46-50%）年化偏低（1-7%）
- **年化>10%的组合胜率集中在44-45%**
- 未找到同时满足"年化>10%且胜率>50%"的组合

### 风控效果
- 止损5-8%略微降低年化（约0.6%），但限制最大回撤
- 止损10%以上反而降低收益（过多止损影响盈利持仓）
- 无风控年化最高但回撤风险更大

## 发现的问题和不方便之处

### 策略Bug
1. **trend_stop**: `_closes = deque(...)` 覆盖整个 dict
2. **breakout_strategy**: `bars["high"]` 索引错误应为 `bars[i]["high"]`
3. **新策略无成交**: 信号生成了但订单没生成，需排查引擎流程

### CLI 不便
1. **backtest run 需完整UUID**: 后台运行时短UUID无效
2. **portfolio list 截断UUID**: 创建后需立即记下完整ID
3. **无 portfolio 详情命令**: 查看映射需Python脚本
4. **component create 生成空模板**: 每次需手动编辑
5. **批量创建回测不稳定**: 部分回测创建失败（shell脚本问题）
6. **ratio_sizer 空占位**: 无按资金比例自动计算手数的功能

### 系统性问题
1. **资金限制**: 100k初始资金，茅台等高价股无法开仓
2. **回测耗时长**: 每个约7-8分钟，参数扫描效率低
3. **信号→订单转换**: 部分策略信号无效，引擎不报错
