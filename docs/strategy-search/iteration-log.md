### 关键发现

1. **主动策略无法超越B&H**: 紫金矿业B&H 3年年化27-42%（取决于仓位），最佳主动策略14.18%，仅捕获35%B&H收益
2. **大仓位≠高收益**: 7500股momentum_trail仅1.91%年化（vs 4200股10.18%），原因是长持策略锁死资金
3. **策略类型分析**:
   - 短期动量(Momentum): 年化最高14.18%，但胜率低42%
   - 动量+ATR止损(MomentumTrail): 胜率高54-60%，但年化低8-10%
   - MA交叉(TrendCapture): 信号太少，不适合单股
   - 金字塔加仓(PyramidTrend): 完全失败

### 最终结论

**在当前Ginkgo框架的组件能力范围内（单股、固定仓位、日线频率），经过39轮迭代测试，无法稳定实现≥20%年化的主动交易策略。**

最高年化记录：**14.45%**（Momentum, th=2%, 7500股, 紫金矿业, 100k资金, frequency=1d）

原因分析：
1. 主动策略的频繁进出错过大趋势（仅捕获B&H的35%收益）
2. 单股日频策略无法利用分散化和日内信号
3. 3年回测期内A股结构性行情明显，少数个股暴涨但策略难以提前识别
4. 固定仓位限制：无法根据波动率或信号强度调整仓位

建议方向（超出当前框架范围）：
- 多股组合+动态仓位分配
- 分钟级/小时级频率
- 机器学习信号增强
- 多因子模型

## 搜索过程总结

### 最佳标的：紫金矿业 (601899.SH)
- 价格：~34元/股，适合大仓位操作
- 波动性：强趋势+适当震荡，适合动量策略
- 其他测试标的：茅台(太贵)、招行(银行震荡)、平安(波动小)、美的(中等)、比亚迪(不适合趋势策略)、赛力斯(亏损)

### 最佳策略类型：短期动量 (Momentum)
- 回溯期 8-10天最优
- 动量阈值 2%最优
- 较短的回溯期捕捉更强的短期趋势信号
- 年化13.3%是最高记录

### 胜率分析
- 动量策略胜率普遍在 32-46% 区间
- **突破**: MomentumTrail策略通过ATR跟踪止损同时实现了≥50%胜率和≥10%年化
- 关键因素: 2.5倍ATR跟踪止损让盈利持仓跑得更远，同时及时截断亏损
- 趋势MA=40天比50天更灵敏，入场更早，捕获更多趋势利润
- 仓位3800-4200股是最佳区间（紫金矿业~15-20元，100k资金）

### MomentumTrail 参数扫描结论
- **trail_mult**: 2.5x最佳（2.0x胜率高但年化低，3.0x年化低且胜率降）
- **threshold**: 2.5%最佳（2.0%太宽松导致大量不盈利交易）
- **trend_period**: 40天最佳（50天稍迟钝，30天太敏感）
- **position_size**: 3800-4200股最佳（太小收益不够，太大资金不够复用）

### 风控效果
- 止损5-8%略微降低年化（约0.6%），但限制最大回撤
- 止损10%以上反而降低收益（过多止损影响盈利持仓）
- 无风控年化最高但回撤风险更大

## 发现的问题和不方便之处

### 策略Bug（不修改源码，记录供参考）
1. **trend_stop**: `_closes = deque(...)` 覆盖整个 dict
2. **breakout_strategy**: `bars["high"]` 索引错误应为 `bars[i]["high"]`
3. **mean_reversion**: `self._prev_rsi = rsi` 覆盖整个 dict，应为 `self._prev_rsi[code] = rsi`
4. **moving_average_crossover**: `self._ma_states = {...}` 同上

### 组件参数问题
- **Python API创建的portfolio参数格式错误**: 通过MParam直接设置时，值不应包含CLI格式的index前缀（如 `0:"value"`），应直接设为 `"value"`。CLI的 `--param '0:value'` 格式会自动解析index和value
- **file_service.update()不生效**: `file_service.update(uuid, data=source)` 返回True但不更新源码，需用 `crud.modify(filters={'uuid': uuid}, updates={'data': source.encode('utf-8')})` 代替（已知bug，本分支修复中）

### CLI 不便
1. **backtest run 需完整UUID**: 后台运行时短UUID无效
2. **portfolio list 截断UUID**: 创建后需立即记下完整ID
3. **无 portfolio 详情命令**: 查看映射需Python脚本
4. **component create 生成空模板**: 每次需手动编辑
5. **批量创建回测不稳定**: 部分回测创建失败（shell脚本问题）
6. **ratio_sizer 空占位**: 无按资金比例自动计算手数的功能

### 系统性问题
1. **资金限制**: 100k初始资金，茅台等高价股无法开仓
2. **回测耗时长**: 每个约7-8分钟，参数扫描效率低
3. **信号→订单转换**: 部分策略信号无效，引擎不报错
